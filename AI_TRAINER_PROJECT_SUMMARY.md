## Executive Summary

**AI Trainer** is a modern web application that generates personalized workout programs using artificial intelligence. The application allows users to input their fitness parameters (target muscle groups, available equipment, injuries, and desired duration) and receive customized workout plans generated by Claude AI. Beyond simple generation, the app includes an interactive workout session tracker with rest timers, set completion tracking, and progress monitoring.

### What Problem Does It Solve?

- **Personalization**: Creates workouts tailored to individual needs, equipment availability, and physical limitations
- **Accessibility**: Makes personal training expertise available to anyone with internet access
- **Tracking**: Provides real-time workout guidance with timers and progress tracking
- **Safety**: Accounts for injuries and limitations to prevent aggravation

## Core Functionality

### 1. Workout Generation

**User Inputs:**
- **Body Parts**: Select up to 3 muscle groups (Chest, Shoulders, Back, Biceps, Triceps, Legs, Glutes, Core)
- **Special Modes**: Full Body or HIIT (exclusive selections)
- **Equipment**: Add any available gym equipment (dumbbells, barbell, resistance bands, etc.)
- **Injuries/Limitations**: Free text input for physical constraints
- **Duration**: Choose from 30, 45, 60, or 90-minute workouts

**AI Processing:**
- Backend constructs a detailed prompt for Claude AI
- AI generates workout with 6-10 exercises targeting specified muscles
- Exercises are ordered logically (compound movements first, then isolation)
- Each exercise includes: sets, reps, rest time, form tips, and YouTube search links

**Output Format:**
- Structured markdown document
- Organized by exercise with clear sections
- Includes warm-up recommendations and cool-down notes

### 2. Workout Display

**Preview Mode (WorkoutDisplay.jsx):**
- Expandable exercise cards
- Click any exercise to reveal full details
- Shows exercise name, sets/reps, rest time, video links, and descriptions
- Users can review the entire workout before starting

### 3. Interactive Workout Session

**Active Training Mode (WorkoutSession.jsx):**
- Parses markdown into structured exercise data
- Displays current exercise with checkboxes for each set
- Rest timer automatically starts after completing a set
- Progress bar shows overall workout completion percentage
- Exercises automatically reorder (incomplete exercises appear first)
- Completion screen with congratulatory message when finished

### 4. Rest Timer

**Timer Component (Timer.jsx):**
- Countdown display in MM:SS format
- Visual progress bar
- Pause/resume functionality
- Skip rest button for advanced users
- Audio notification when rest period ends
- Color changes when less than 10 seconds remain

**Purpose:** Provides interactive workout tracking with real-time progress monitoring

5. **Exercise Sorting:**

6. **Completion Screen:**

**State Variables:**

#### workoutParser.js (135 lines) - Markdown Processing

**Purpose:** Converts AI-generated markdown into structured JavaScript objects

**Function 1: `parseWorkoutMarkdown(markdown)`**

Extracts structured data from markdown text:

```javascript
Input: Markdown string
Output: Array of exercise objects

Exercise Object Structure:
{
  name: String,           // Exercise name (from ### headings)
  sets: Number,           // Number of sets (default: 3)
  reps: String,          // Rep range (e.g., "8-12", default: "8-12")
  rest: Number,          // Rest seconds (default: 60)
  videoLink: String,     // YouTube search URL
  description: String    // Exercise details and form tips
}
```

**Parsing Logic:**
- Identifies exercises by `###` markdown headings
- Extracts sets from bullet points: `* Sets: 3`
- Extracts reps from patterns: `* Reps: 8-12` or `* 3 sets × 8-12 reps`
- Extracts rest time: `* Rest: 60 seconds`
- Captures video links from markdown URL syntax: `[text](url)`
- Compiles remaining bullet points as description

**Function 2: `calculateWorkoutStats(exercises)`**

Calculates workout metrics:

```javascript
Input: Array of exercise objects
Output: {
  totalExercises: Number,
  totalSets: Number,
  estimatedDuration: Number  // in minutes
}
```

**Function 3: `createSessionState(exercises)`**

Initializes tracking arrays for workout session:

```javascript
Input: Array of exercise objects
Output: 2D array of booleans (all false initially)

Example: [
  [false, false, false],  // Exercise 1: 3 incomplete sets
  [false, false, false, false]  // Exercise 2: 4 incomplete sets
]
```

## Data Flow & Processing

### Complete Workflow

```
1. USER INPUT PHASE
   ├─ User selects body parts (Main.jsx)
   ├─ User adds equipment (Main.jsx)
   ├─ User specifies injuries (Main.jsx)
   ├─ User chooses duration (Main.jsx)
   └─ User clicks "Generate Workout" (Main.jsx)

2. API REQUEST PHASE
   ├─ training() function called (ai.js)
   ├─ POST request to /api/training
   ├─ Backend receives parameters (app.js)
   └─ System prompt + user parameters sent to Claude

3. AI GENERATION PHASE
   ├─ Claude processes requirements
   ├─ Generates exercise list
   ├─ Formats as markdown
   └─ Returns to backend

4. RESPONSE PHASE
   ├─ Backend sends markdown to frontend
   ├─ Main.jsx stores in workoutData state
   └─ UI switches to Workout component

5. DISPLAY PHASE (Preview)
   ├─ WorkoutDisplay.jsx receives markdown
   ├─ react-markdown renders content
   ├─ User reviews workout
   └─ User clicks "Start Workout"

6. SESSION PHASE (Active Tracking)
   ├─ WorkoutSession.jsx parses markdown (parseWorkoutMarkdown)
   ├─ Creates completion tracking arrays (createSessionState)
   ├─ Displays first exercise with sets
   └─ User begins workout

7. EXERCISE EXECUTION LOOP
   ├─ User performs set
   ├─ User checks off completed set
   ├─ Timer component starts rest countdown
   ├─ Timer plays sound when complete
   ├─ User proceeds to next set/exercise
   └─ Repeat until all sets complete

8. COMPLETION PHASE
   ├─ All sets marked complete
   ├─ Completion screen displays
   ├─ User can end session
   └─ Returns to input form for new workout
```

## AI Integration

### Claude API Configuration

**Model Selection:** Claude 3 Haiku

**Reasons for Haiku:**
- Cost-effective for high-volume requests
- Fast response times (3-5 seconds)
- Sufficient fitness knowledge
- Good markdown formatting capabilities

**Alternative Considerations:**
- Claude 3 Sonnet: More detailed explanations (higher cost)
- Claude 3 Opus: Maximum quality (highest cost, slower)

### Prompt Engineering

**System Prompt Strategy:**

The backend uses a carefully crafted system prompt that:

1. **Defines Role:** "You are a professional personal trainer"
2. **Sets Constraints:**
   - ONLY target specified body parts
   - ONLY use provided equipment
   - AVOID injury-aggravating exercises
3. **Specifies Format:**
   - Markdown with ### headings
   - Bullet points for sets/reps/rest
   - YouTube search links (not direct links)
4. **Ensures Quality:**
   - Compound movements first
   - Form tips included
   - Modifications for beginners
   - Concise descriptions

**User Prompt Construction:**

```javascript
`Create a ${duration}-minute workout targeting ${bodyPart.join(', ')}.

Available equipment: ${equipmentArr.join(', ')}
Injuries/limitations: ${injury || 'None'}

Generate the workout following all guidelines.`
```

### Response Processing

**Expected Output Structure:**

```markdown
# 45-Minute Chest & Triceps Workout

## Warm-up
* 5 minutes light cardio
* Dynamic stretches

### 1. Dumbbell Bench Press
* Sets: 4
* Reps: 8-10
* Rest: 90 seconds
* [Video Tutorial](YouTube search: "dumbbell bench press Jeff Nippard")
* Compound movement for overall chest development
* Keep shoulder blades retracted, lower dumbbells to chest level

### 2. Incline Dumbbell Press
* Sets: 3
* Reps: 10-12
* Rest: 75 seconds
* [Video Tutorial](YouTube search: "incline dumbbell press AthleanX")
* Targets upper chest
* Maintain 30-45 degree incline

...

## Cool-down
* Static stretches for chest and triceps
* 5 minutes
```

### Error Handling

**API Failures:**
- Network errors: Display user-friendly message
- Invalid API key: Log server-side error
- Rate limiting: Retry logic (future enhancement)
- Malformed responses: Fallback to error state
